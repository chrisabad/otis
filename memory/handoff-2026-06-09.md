**Key lesson:** Two FK columns had no indexes (`heartbeat_runs.retry_of_run_id`, `heartbeat_runs.wakeup_request_id`) — each DELETE was doing full seq scans → hours per run. Added indexes first, then retry succeeded in minutes.

### 3. Advisory lock fix
Recurring pattern: Paperclip dispatch serializes via `pg_advisory_xact_lock`. Zombie `idle in transaction` sessions hold the lock indefinitely → dispatch queue piles up → UI sluggish. Fixed permanently via:
```sql
ALTER SYSTEM SET idle_in_transaction_session_timeout = '2min';
ALTER SYSTEM SET statement_timeout = '2min';
SELECT pg_reload_conf();
```
For intentional long ops (bulk DELETEs etc.) override inline: `SET statement_timeout = 0`.
